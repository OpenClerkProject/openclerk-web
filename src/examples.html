Citation Checker — check one or more citations fast

Best for: quickly checking a citation or two, no document involved.

  1. Pick a Bluebook edition from the dropdown.
  2. Paste one or more citations into the text box, or upload a .txt/.docx/.odt file.
  3. Click Check citations (or press Ctrl/Cmd+Enter).

Try pasting this — one clean citation and one with a real, flagged formatting issue:

Ashcroft v. Iqbal, 556 U.S. 662, 678 (2009)
Smith v. Acme Company, 123 U.S. 456 (1990)
Open the Citation Checker →

The second citation is intentionally flagged: Bluebook's Table T6 abbreviates "Company" to "Co." in a case name, so the checker warns instead of passing it silently.

Document Editor — work with a whole document

Best for: a full brief or memo, on any screen size (including mobile).

  1. Paste your document's text directly into the document pane, or load a .txt/.docx/.odt/.pdf file (try the example PDF above).
  2. Manage Hyperlinks — look citations up against a real provider (e.g. CourtListener) and hyperlink only the ones that resolve, or hand-assign URLs to other citations.
  3. Bluebook Check — checks mechanical formatting conventions and lists every issue found.
  4. Find Hallucinations — checks every case citation against the platforms you choose, in order, and flags any that don't resolve on any of them.
  5. Embed Cited Text — pulls in the actual cited page's text next to a pincite citation, so you can verify the quote without leaving the document.
  6. Format with the toolbar (bold/italic/underline, headings, lists), then Download as .txt or .odt.

OpenClerk Studio — the same tools, a richer desktop layout

Best for: a larger screen (860px+) and a more desktop-app-like editing session.

Studio runs the exact same Manage Hyperlinks / Bluebook Check / Find Hallucinations / Embed Cited Text workflows as the Document Editor above, in a fuller layout: a document outline you can click to jump between sections, a comment gutter that surfaces flagged citations next to the text as you work, and a status bar with a live word count and citation-health summary. Its File menu can load the example PDF above the same way, and its Insert menu adds a generic "link this text to a URL" command for anything that isn't a case citation.

PDF & OCR Tools — extract and check citations from a PDF

Best for: a PDF you don't want to retype, including scanned documents with no text layer.

  1. Upload a PDF (try the example above — it has no embedded text layer, so it's a real test of the OCR fallback).
  2. Optionally check the box to verify citations against CourtListener and flag any that don't resolve.
  3. Click Extract & check citations.

Connecting a citation lookup provider (optional)

Manage Hyperlinks, Find Hallucinations, Embed Cited Text, and PDF & OCR Tools' verification checkbox all need a lookup provider to check a citation against real case law. Today that's CourtListener, a free service from the Free Law Project — sign up for a free account and grab an API token from courtlistener.com/profile/api-token, then paste it into any page's Manage Hyperlinks panel. Only the matched citation strings are ever sent to CourtListener — never your document's text.
